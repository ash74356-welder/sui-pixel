# 广州像素画布项目规格说明书

## 项目概述

**项目名称**: 广州像素画布 (GZ Pixel Canvas)  
**项目类型**: Web 应用  
**核心功能**: 基于广州地图的匿名多人协作像素绘制平台  

---

## 1. 项目目标

### 1.1 主要目标
- 创建一个以广州地图为背景的多人协作像素绘制网站
- **无需注册登录**，连接即可绘制，使用 IP 区分用户
- 像素点支持自定义大小，**仅正方形像素**（无缝铺满地图）
- 支持导入图片/文字并像素化后添加到地图
- 支持多层级缩放，不同缩放级别显示不同精细度的地图
- **像素过多时自动隐藏像素层，仅显示地图背景**
- 多用户实时协作，所有用户绘制的像素同时显示
- 鼠标滚轮控制地图缩放
- **后来者覆盖先绘制的像素**，每个像素点只保留一个颜色值

### 1.2 目标用户
- 广州本地居民
- 地图爱好者
- 像素艺术创作者
- 社区协作参与者

## 3. 像素与地图对应关系

### 3.1 最小像素单元定义

| 参数 | 值 | 说明 |
|------|-----|------|
| 最小像素尺寸 | 约 10-20 米 | 对应一个小街道的大小 |
| 像素网格精度 | 0.0001 度 | 约 10 米 (经纬度) |
| 像素视觉大小 | 固定 8-16px | 不随地图缩放改变 |
| **像素形状** | **仅正方形** | 无缝铺满地图 |

### 3.2 像素大小等级

| 等级 | 实际尺寸 | 视觉尺寸 | 适用场景 |
|------|----------|----------|----------|
| 小 (S) | ~10m | 8px | 精细细节、小标记 |
| 中 (M) | ~20m | 12px | 常规绘制 |
| 大 (L) | ~40m | 16px | 大面积填充 |

### 3.3 像素层显示策略

| 场景 | 显示方式 | 说明 |
|------|----------|------|
| 像素数量 < 阈值 | 显示像素层 | 正常显示所有像素 |
| 像素数量 >= 阈值 | **隐藏像素层** | **仅显示地图背景** |
| 用户主动绘制 | 临时显示 | 绘制时显示新像素 |

**阈值设定**: 根据性能测试确定，如 50,000 像素

## 4. 技术架构

### 4.1 技术栈

```
前端:
├── Leaflet.js (地图引擎)
├── HTML5 Canvas (像素绘制层)
├── Socket.io-client (实时通信)
├── Vanilla JavaScript / TypeScript
└── CSS3 / Tailwind CSS

├── Node.js + Express (Web服务器)
├── Socket.io (WebSocket 实时通信)

└── 用户识别: IP 地址
```

## 5. 数据模型

### 5.1 用户会话 (基于 IP)

```typescript
interface UserSession {
  id: string;               // 会话唯一标识 (基于 IP 生成)
  ip: string;               // IP 地址
  ipHash: string;           // IP 哈希 (隐私保护)
  displayName: string;      // 显示名称 (如 "用户 192.168")
  connectedAt: Date;        // 连接时间
  lastActivityAt: Date;     // 最后活动时间
  pixelCount: number;       // 该用户绘制的像素数
}
```

### 5.2 像素数据

```typescript
interface Pixel {
  id: string;               // 唯一标识
  lat: number;              // 纬度
  lng: number;              // 经度
  gridX: number;            // 网格 X 坐标
  gridY: number;            // 网格 Y 坐标
  color: string;            // 颜色 (经典 256 色 hex)
  size: 'small' | 'medium' | 'large';  // 像素大小
  **shape: 'square'**;      // **仅正方形**
  authorId: string;         // 作者会话 ID (IP 标识)
  authorName: string;       // 作者显示名称
  createdAt: Date;          // 创建时间
  updatedAt: Date;          // 更新时间 (覆盖时更新)
}
```

**注意**: 同一 gridX/gridY 位置只保留一个 Pixel 记录，后来者覆盖更新。

### 5.3 地图配置

```typescript
interface MapConfig {
  center: [number, number];     // 中心坐标 [lat, lng] - 广州
  defaultZoom: number;          // 默认缩放级别
  minZoom: number;              // 最小缩放
  maxZoom: number;              // 最大缩放
  pixelGridSize: number;        // 像素网格大小 (度)
  maxDrawSize: number;          // 单次最大绘制尺寸 (400)
  cooldownSeconds: number;      // 冷却时间 (秒)
  **pixelDisplayThreshold: number**; // **像素显示阈值**
}
```

## 6. 核心交互流程

### 6.1 像素层显示控制流程

```
地图缩放或移动
    ↓
计算当前视口内像素数量
    ↓
判断像素数量是否超过阈值?
    ↓
    ├── 是 → **隐藏 Canvas 像素层** → 仅显示地图
    └── 否 → 显示 Canvas 像素层 → 正常渲染像素
    ↓
用户主动绘制新像素
    ↓
临时显示新绘制的像素
```

### 6.2 图片导入像素化流程（独立模块）

```
用户选择图片文件
    ↓
**格式验证** (JPG, PNG, GIF, WebP, BMP, TIFF)
    ↓
上传至服务器
    ↓
**独立图片处理模块**:
    - 读取图片元数据
    - **高精度缩放** (支持多种算法: 最近邻/双线性/Lanczos)
    - **颜色量化** (256色，支持多种算法)
    - **生成多种规格** (原比例/固定宽度/固定高度/固定尺寸)
    ↓
返回多种规格预览
    ↓
用户选择规格和尺寸
    ↓
确认放置到地图 (限制 400x400)
    ↓
批量 UPSERT 像素点
    ↓
广播给所有用户
```

## 9. 图片转换模块规格

### 9.1 支持格式

| 格式 | 读取 | 说明 |
|------|------|------|
| JPEG/JPG | ✅ | 有损压缩，适合照片 |
| PNG | ✅ | 无损压缩，支持透明 |
| GIF | ✅ | 支持动画（取第一帧） |
| WebP | ✅ | 现代格式，高压缩率 |
| BMP | ✅ | 无压缩，文件较大 |
| TIFF | ✅ | 专业图像格式 |

### 9.2 转换规格

| 规格类型 | 说明 |
|----------|------|
| 原比例缩放 | 保持宽高比，缩放到指定宽度或高度 |
| 固定宽度 | 指定宽度，高度自动计算 |
| 固定高度 | 指定高度，宽度自动计算 |
| 固定尺寸 | 强制指定宽高（可能变形） |
| 裁剪模式 | 居中裁剪到指定尺寸 |

### 9.3 缩放算法

| 算法 | 适用场景 |
|------|----------|
| 最近邻 | 速度快，保留硬边缘 |
| 双线性 | 平衡速度和质量 |
| Lanczos | 高质量，适合照片 |

### 9.4 颜色量化算法

| 算法 | 说明 |
|------|------|
| 中位切分 | 经典算法，效果均衡 |
| K-Means | 质量高，速度慢 |
| 八叉树 | 速度快，适合实时 |
